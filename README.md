<p>자주 사용하는 깃 명령어 모음</p>
<h2><a id="user-content-구조" class="anchor" aria-hidden="true" href="#구조"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>구조</h2>
<p>코드는 아래 세 단계에 걸쳐 저장된다.</p>
<p>스테이징 -&gt; 커밋 -&gt; 원격저장소</p>
<ol>
<li>git add {파일명} 으로 파일을 스테이징 상태에 넣는다.</li>
<li>git commit 으로 스테이징 상태에 있는 모든 변경사항을 커밋한다. 여기까지가 로컬에서의 작업</li>
<li>git push 로 커밋된 저장소를 원격 저장소로 밀어넣는다.</li>
</ol>

<pre><code>git init
</code></pre>

</code></pre>

<pre><code>git status // 
</code></pre>

</code></pre>

<h2><a id="user-content-commit" class="anchor" aria-hidden="true" href="#commit"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Commit</h2>
<p>커밋 합치기</p>
<pre><code>git rebase -i HEAD~4 // 최신 4개의 커밋을 하나로 합치기
</code></pre>
<p>커밋 메세지 수성</p>
<pre><code>$ git commit --amend // 마지막 커밋메세지 수정(ref)
</code></pre>
<p>간단한 commit방법</p>
<pre><code>$ git add {변경한 파일병}
$ git commit -m “{변경 내용}"
</code></pre>
<p>커밋 이력 확인</p>
<pre><code>$ git log // 모든 커밋로그 확인
$ git log -3 // 최근 3개 커밋로그 확인
$ git log --pretty=oneline // 각 커밋을 한 줄로 표시
</code></pre>
<p>커밋 취소</p>
<pre><code>$ git reset HEAD^ // 마지막 커밋 삭제
$ git reset --hard HEAD // 마지막 커밋 상태로 되돌림
$ git reset HEAD * // 스테이징을 언스테이징으로 변경, ref
</code></pre>
<h2><a id="user-content-branch" class="anchor" aria-hidden="true" href="#branch"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Branch</h2>
<p>master 브랜치를 특정 커밋으로 옮기기</p>
<pre><code>git checkout better_branch
git merge --strategy=ours master    # keep the content of this branch, but record a merge
git checkout master
git merge better_branch            # fast-forward master up to the merge
</code></pre>
<p>브랜치 목록</p>
<pre><code>$ git branch // 로컬
$ git branch -r // 리모트 
$ git branch -a // 로컬, 리모트 포함된 모든 브랜치 보기
</code></pre>
<p>브랜치 생성</p>
<pre><code>git branch new master // master -&gt; new 브랜치 생성
git push origin new // new 브랜치를 리모트로 보내기
</code></pre>
<p>브랜치 삭제</p>
<pre><code>git branch -D {삭제할 브랜치 명} // local
git push origin :{the_remote_branch} // remote
</code></pre>
<p>빈 브랜치 생성</p>
<pre><code>$ git checkout --orphan {새로운 브랜치 명}
$ git commit -a // 커밋해야 새로운 브랜치 생성됨
$ git checkout -b new-branch // 브랜치 생성과 동시에 체크아웃
</code></pre>
<p>리모트 브랜치 가져오기</p>
<pre><code>$ git checkout -t origin/{가져올 브랜치명} // ref
</code></pre>
<p>브랜치 이름 변경</p>
<pre><code>$ git branch -m {new name} // ref
</code></pre>

<p>태그 생성</p>
<pre><code>git tag -a {tag name} -m {tag message} {commit hash}
git tag {tag name} {tag name} -f -m "{new message}" // Edit tag message
</code></pre>
<p>태그 삭제</p>

<h2><a id="user-content-기타" class="anchor" aria-hidden="true" href="#기타"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>기타</h2>
<p>파일 삭제</p>
<pre><code>git rm --cached --ignore-unmatch [삭제할 파일명]
</code></pre>
<p>히스토리 삭제</p>
<ul>
<li>목적: 패스워드, 아이디 같은 비공개 정보가 담긴 파일을 실수로 올렸을 때 삭제하는 방법이다. (history에서도 해당 파일만 삭제)</li>
</ul>
<pre><code>$ git clone [url] # 소스 다운로드
$ cd [foler_name] # 해당 폴더 이동
$ git filter-branch --index-filter 'git rm --cached --ignore-unmatch [삭제할 파일명]' --prune-empty -- --all # 모든 히스토리에서 해당 파일 삭제
$ git push origin master --force # 서버로 전송
</code></pre>
<p>히스토리에서 폴더 삭제:</p>
<pre><code>git filter-branch --tree-filter 'rm -rf vendor/gems' HEAD
</code></pre>

$ git pull upstream {브랜치명}
</code></pre>

<pre><code>$ git gc
